Big Data Processing with Apache Airflow

As more companies practices data collecting technology in the last decades, big data is widely used in many industries. Apache Software Foundation has developed something called "Airflow". What is a 'workflow' in this case? It's pretty much what it sounds like. Airflow is an open-source workflow management platform that has a sequence of tasks that might launch on a schedule or perhaps on an event. It is also something that is used to build data pipelines. It is really a pretty general problem that applies to many things.

In this project I will use airflow to create a data pipeline for big data.

First, we need to download airflow yaml file from Apache through terminal window and use the curl command ("curl 'https://airflow.apache.org/docs/apache-airflow/2.1.1/docker-compose.yaml' -o docker-compose.yaml"). This allows us to run airflow in docker container.

This enables us to run airflow on our local browser

As the picture above shows that there is nothing under DAGs (Directed Acyclic Graph), we need to create some python files under this local folder. As the yaml file composes, airflow generates 3 folders: DAGs, logs, and plugins. Here we will make a python file under dags and include 'python_square_operator', which automates in a time interval and squares a number.

As the result, we can see python_square_operator from Airflow.

Start the DAG by toggling the switch to the left of its name. Select the DAG and open up the Graph View. According to the Logs, we can see that airflow successfully returned the square of 13, which is 169. This is correct as we see the number we put in the python file under DAGs is 13 with the function of doubling the number. Airflow also marked the task as success
